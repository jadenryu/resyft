üîß PHASE 2: BACKEND DEPLOYMENT
=============================

‚ñ° 2.1 Choose Backend Hosting Platform
   RECOMMENDED OPTIONS:
   - Railway (Easiest, $5/month)
   - Render (Free tier available)
   - DigitalOcean App Platform ($5/month)
   - AWS/GCP (More complex, scalable)

‚ñ° 2.2 Deploy Python Backend
   For Railway (Recommended):
   - Install Railway CLI: npm install -g @railway/cli
   - Login: railway login
   - Deploy: railway up
   - Set environment variables in Railway dashboard
   - Get your backend URL (e.g., https://your-app.railway.app)

‚ñ° 2.3 Configure Backend
   - Update CORS settings to allow your domain
   - Set up production database
   - Configure all API keys and secrets
   - Test API endpoints work correctly

‚ñ° 2.4 Backend Health Check
   - Test all API endpoints
   - Verify database connections
   - Check logs for errors
   - Document your backend URL for frontend use

üíª PHASE 3: FRONTEND DEPLOYMENT
==============================

‚ñ° 3.1 Update Frontend Configuration
   - Update .env.production with backend URL
   - Set NEXT_PUBLIC_API_URL to your backend URL
   - Configure any other environment variables

‚ñ° 3.2 Choose Frontend Hosting Platform
   RECOMMENDED OPTIONS:
   - Vercel (Best for Next.js, free tier)
   - Netlify (Great alternative, free tier)
   - AWS Amplify (More advanced)

‚ñ° 3.3 Deploy to Vercel (Recommended)
   - Install Vercel CLI: npm install -g vercel
   - Login: vercel login
   - Deploy: vercel --prod
   - Set environment variables in Vercel dashboard
   - Get your app URL (e.g., https://resyft.vercel.app)

‚ñ° 3.4 Alternative: Deploy to Netlify
   - Build project: npm run build
   - Drag build folder to Netlify dashboard
   - Or connect GitHub repository
   - Configure environment variables
   - Get your app URL

üåê PHASE 4: DOMAIN CONFIGURATION
===============================

‚ñ° 4.1 Get DNS Records from Hosting Platform
   For Vercel:
   - Go to Project ‚Üí Settings ‚Üí Domains
   - Add your domain (e.g., resyft.com)
   - Copy the provided DNS records

‚ñ° 4.2 Configure Namecheap DNS
   - Login to Namecheap account
   - Go to Domain List ‚Üí Manage your domain
   - Click "Advanced DNS"
   - Delete existing A/CNAME records
   - Add new records from Vercel:
     * A Record: @ ‚Üí Vercel IP address
     * CNAME: www ‚Üí your-app.vercel.app

‚ñ° 4.3 SSL Certificate Setup
   - Usually automatic with Vercel/Netlify
   - Verify HTTPS works after DNS propagation
   - Force HTTPS redirects in hosting platform

‚ñ° 4.4 DNS Propagation Wait
   - Wait 2-24 hours for DNS changes to propagate
   - Check progress: whatsmydns.net
   - Test your domain periodically

üîê PHASE 5: SECURITY & ENVIRONMENT SETUP
=======================================

‚ñ° 5.1 Production Environment Variables Checklist
   Frontend (.env.production):
   - NEXT_PUBLIC_API_URL=https://your-backend.railway.app
   - NEXT_PUBLIC_SITE_URL=https://your-domain.com
   - NEXTAUTH_URL=https://your-domain.com (if using auth)
   - NEXTAUTH_SECRET=random-secret-key
   - Any analytics keys (Google Analytics, etc.)

   Backend Environment Variables:
   - DATABASE_URL=your-production-database
   - OPENAI_API_KEY=your-openai-key
   - ANTHROPIC_API_KEY=your-anthropic-key
   - CORS_ORIGINS=https://your-domain.com
   - SECRET_KEY=random-secret-for-backend

‚ñ° 5.2 Security Configuration
   - Enable HTTPS everywhere
   - Configure CORS properly
   - Set up rate limiting
   - Enable security headers
   - Never commit secrets to GitHub

‚ñ° 5.3 Database Setup (if applicable)
   - Set up production database (PostgreSQL recommended)
   - Run migrations
   - Set up backups
   - Configure connection pooling

üÜò TROUBLESHOOTING COMMON ISSUES
===============================

Issue: "API calls failing in production"
Solution: Check CORS settings, verify environment variables, check network tab in browser

Issue: "Domain not working after 24 hours"
Solution: Verify DNS records match exactly, check TTL settings, contact Namecheap support

Issue: "Site loading slowly"
Solution: Enable caching, optimize images, use CDN, check server resources

Issue: "SSL certificate errors"
Solution: Usually auto-resolves, force SSL renewal in hosting dashboard

Issue: "500 server errors"
Solution: Check backend logs, verify database connections, check environment variables
